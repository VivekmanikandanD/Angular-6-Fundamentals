Life Cycle hooks

Lifecycle sequence

  1. ngOnChanges();
  2. ngOnInit();
  3. ngDoCheck();
  4. ngOnAfterContentInit();
  5. ngOnAfterContentChecked();
  6. ngOnViewInit();
  7. ngOnViewChecked();
  8. ngOnDestroy();

Description

ngOnChanges()

Angular calls its ngOnChanges() method whenever it detects changes to input properties of the component (or directive)(@Input)

ngOnInit()
1.To perform complex initializations shortly after construction.
2.To set up the component after Angular sets the input properties.
ngDoCheck()
1.To detect and act upon changes that Angular doesn't catch on its own.
2.To detect a change that Angular overlooked.
ngOnAfterContentInit()

Angular calls after projecting external content into the child component between ng-content tags from after-content tags in the parent component. The ng-content is a placeholder for the external content We should never put content between a component's element tags unless you intend to project that content into the component. Angular completes composition of the projected content before finishing the composition of this component's view. There is a small window between the AfterContent... and AfterView... hooks to modify the host view.

ngOnAfterContentChecked()

Angular calls after projecting external content into the child component between ng-content tags from after-content tags in the parent component. The ng-content is a placeholder for the external content We should never put content between a component's element tags unless you intend to project that content into the component. Angular completes composition of the projected content before finishing the composition of this component's view. There is a small window between the AfterContent... and AfterView... hooks to modify the host view.

ngOnViewInit()

Angular calls after it creates a component's child views. Child view changes can be detected by querying for the child view via the property decorated with @ViewChild.

ngOnViewChecked()

Angular calls after it creates a component's child views. Child view changes can be detected by querying for the child view via the property decorated with @ViewChild.

ngOnDestroy()

we should put cleanup logic in ngOnDestroy(), the logic that must run before Angular destroys the directive.Place to free resources. Unsubscribe from Observables and DOM events.
